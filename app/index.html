Założenia

  • Aplikacja korzysta z dowolnego publicznego API i wyświetla listę podanych przez użytkownika projektów, pobranych elementów (publiczne API dowolne, np. API Travis CI/Github – status ostatnich udanych buildów wybranych projektów)
  • Aplikacja przewiduje, że użytkownik poda niepoprawny url
  • Lista składa się z max. 5 elementów
  • UI: tytuł i założenie projektu, lista, autor. Co drugi element listy wyróżniony innym kolorem
  • Użycie Less, Flexbox
  • Kod znajduje się na publicznym repozytorium Github’a
